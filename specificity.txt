CSS Specificity Explanation:

Paragraph A (matches only p - element selector):
- color: gray (element selector p)
- font-size: 1rem (element selector p)
- font-family: Georgia, serif (element selector p)
- line-height: 1.4 (element selector p)
- padding: 4px 0 (element selector p)

Paragraph B (matches p and .info):
- color: steelblue (class selector .info)
- font-size: 1.25rem (class selector .info)
- font-family: Arial, sans-serif (class selector .info)
- background-color: #e8f4fd (class selector .info)
- padding: 8px 14px (class selector .info)
- border-left: 5px solid steelblue (class selector .info)
- line-height: 1.4 (element selector p - not overridden)

Paragraph C (matches p and #warning):
- color: darkred (id selector #warning)
- font-size: 0.8rem (id selector #warning)
- font-family: 'Courier New', monospace (id selector #warning)
- font-style: italic (id selector #warning)
- text-transform: uppercase (id selector #warning)
- letter-spacing: 0.15em (id selector #warning)
- border: 2px dashed darkred (id selector #warning)
- padding: 10px (id selector #warning)
- background-color: #fff0f0 (id selector #warning)
- line-height: 1.4 (element selector p - not overridden)

Paragraph D (matches p, .info, #warning, and #warning.info):
- color: darkorange (id+class selector #warning.info - highest specificity)
- font-family: 'Courier New', monospace (id selector #warning - #warning.info doesn't define it)
- background-color: #fff7e0 (id+class selector #warning.info)
- border: 2px solid darkorange (id+class selector #warning.info)
- font-size: 1.1rem (id+class selector #warning.info)
- font-style: normal (id+class selector #warning.info)
- letter-spacing: 0.03em (id+class selector #warning.info)
- text-transform: uppercase (id selector #warning - not overridden)
- padding: 10px (id selector #warning - not overridden)
- line-height: 1.4 (element selector p - not overridden)

For Paragraph D, all four selectors define color — the browser uses darkorange from #warning.info because it has the highest specificity (0,1,1,0), compared to:
- p: 0,0,0,1
- .info: 0,0,1,0
- #warning: 0,1,0,0

For font-family, #warning.info does not define it, so the rule from #warning wins (specificity 0,1,0,0).

For the h1 element:
- External CSS (style.css): h1 { color: blue; font-style: italic; padding: 10px; }
- Internal CSS: h1 { color: red; }

Since internal CSS has higher precedence in the cascade than external CSS (both have specificity 0,0,0,0 but internal appears later), the h1 element will have color: red from internal CSS, and font-style: italic and padding: 10px from external CSS.
